D. FEATURES HIỆN CÓ

CLI Commands:
  bridge-cli create-agent <name> <project> [--purpose] [--model]
  bridge-cli delete-agent <name>
  bridge-cli list-agents
  bridge-cli dispatch <agent> "<task>" [--channel] [--chat-id] [--msg-id]
  bridge-cli kill <agent>
  bridge-cli queue [--agent]
  bridge-cli cancel <task-id>
  bridge-cli cost [--agent] [--days]
  bridge-cli status [--agent]
  bridge-cli setup                     (wizard: channel/mcp mode, token, pairing)
  bridge-cli setup-telegram            (config Telegram token)
  bridge-cli doctor                    (health check: deps, config, hooks, connectivity)
  bridge-cli daemon install|uninstall|start|stop|status|logs
  bridge-cli create-team <name> <agents...>
  bridge-cli team-dispatch <team> "<task>"
  bridge-cli set-model <agent> <model>
  bridge-cli setup-cron / remove-cron
  bridge-cli memory <agent>
  bridge-cli --version

Channels:
  - channel mode: Node.js/Bun server (channel/) — Telegram grammy bot + MCP tools
  - mcp mode: Python MCP server (mcp_server.py) — native Claude Code MCP integration

Capabilities:
  - Multi-session dispatch: mỗi agent = project riêng, session_id = agent+project
  - Worktree isolation: mỗi task chạy trong git worktree riêng biệt
  - Stop hook tự động: on_complete.py cập nhật SQLite khi task xong
  - Auto Memory: Claude Code auto-learn patterns, đọc qua /memory command
  - Daemon install: systemd (Linux) / launchd (macOS) background service
  - Telegram file receiving: hỗ trợ nhận file từ Telegram (INBOX_DIR)
  - Permission relay: bridge bot relay permissions về Telegram
  - Team dispatch: broadcast task đến nhiều agents
  - CLAUDE_BRIDGE_HOME: custom data directory
  - install.sh: one-liner curl install script
  - CHANGELOG.md + bun.lock + .env.example + mermaid architecture diagram
