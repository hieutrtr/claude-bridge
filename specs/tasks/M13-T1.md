## What This Task Does
Create the Bridge MCP server skeleton with stdio transport. Server starts, registers tools, and shuts down cleanly. No actual tool implementations yet — just the framework.

## Key Architecture Constraints
- stdio transport (standard for Claude Code MCP servers)
- Python stdlib + `mcp` SDK (only external dependency)
- Server name: "bridge"
- Clean shutdown on EOF/SIGINT
- Must be runnable as `python3 -m claude_bridge.mcp_server`

## Test Plan
Happy path:
- Server initializes with name "bridge"
- Server registers tool list (empty or placeholder)
- Server handles list_tools request

Edge cases:
- Server handles unknown tool call gracefully
- Server handles malformed requests

Errors:
- Server shuts down cleanly on EOF
