## What This Task Does
Design how a lead agent decomposes complex tasks into sub-tasks and dispatches them to teammate agents. Define the data model (teams table, sub-task tracking via parent_task_id), communication flow, and result aggregation.

## Key Architecture Constraints
- Session = Agent + Project (existing model unchanged)
- SQLite with WAL mode, foreign keys ON
- Lead agent dispatches sub-tasks via bridge-cli (using Bash tool from within Claude Code)
- Sub-tasks queue like any other task (reuses Phase 2 queue system)
- Teams are metadata — each agent remains independent with its own session
- CASCADE delete: deleting a team does NOT delete agents

## Design: Team Coordination Protocol

### Data Model

```sql
CREATE TABLE IF NOT EXISTS teams (
    name TEXT PRIMARY KEY,
    lead_agent TEXT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (lead_agent) REFERENCES agents(name)
);

CREATE TABLE IF NOT EXISTS team_members (
    team_name TEXT NOT NULL REFERENCES teams(name) ON DELETE CASCADE,
    agent_name TEXT NOT NULL,
    PRIMARY KEY (team_name, agent_name)
);
```

Tasks table extension:
- Add `parent_task_id INTEGER REFERENCES tasks(id)` — links sub-tasks to parent
- Add `task_type TEXT DEFAULT 'standard'` — values: 'standard', 'team'

### Communication Flow

```
User: team-dispatch fullstack "build user profile page"
  → Bridge creates parent task (type='team') for lead agent
  → Lead agent prompt augmented with:
    - Original task
    - Available teammates + their purposes
    - Instructions to decompose and dispatch via bridge-cli
  → Lead uses Bash tool: bridge-cli dispatch <teammate> "<sub-task>"
  → Sub-tasks created with parent_task_id = parent task ID
  → Sub-tasks queue/run per normal dispatch logic
  → on-complete checks: all sub-tasks done? → notify lead to aggregate
  → Lead produces final summary
  → Parent task marked done
```

### Result Aggregation
- on-complete.py detects sub-task completion
- Checks if all sibling sub-tasks for same parent are done
- If all done: sends aggregation prompt to lead agent
- Lead reads sub-task results and produces final summary
- Total cost = lead cost + sum(sub-task costs)

## Test Plan
Happy path:
- Create team with lead + members, verify DB rows
- List teams shows correct structure
- Delete team removes team but preserves agents

Edge cases:
- Team with single member
- Lead agent same as member (should error)
- Agent referenced in team doesn't exist (should error)
- Delete team with tasks in progress

Errors:
- Duplicate team name
- Empty team name
- Non-existent agent references

## Gaps in Existing Code
- No `teams` or `team_members` tables in schema
- No `parent_task_id` or `task_type` columns in tasks table
- No team-related CLI commands
- on-complete.py has no sub-task awareness
