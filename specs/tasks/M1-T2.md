## What This Task Does
Verify SQLite schema is correct, idempotent, and all CRUD operations work with proper constraints.

## Key Architecture Constraints
- WAL mode enabled
- Foreign keys ON with CASCADE delete (agents → tasks)
- Primary key: (name, project_dir) for agents table
- session_id is UNIQUE
- Default state for new agents is 'created'
- Indexes on tasks.status, tasks.session_id, and unreported tasks

## Test Plan
Happy path:
- Schema init is idempotent (call twice, no error)
- WAL mode is active
- Foreign keys are enforced
- Create agent with all fields
- Create task linked to agent
- Query operations: get_agent, list_agents, get_running_task, get_task_history
- Update operations: update_agent_state, update_task, increment_agent_tasks

Edge cases:
- Duplicate agent name raises IntegrityError
- Duplicate session_id raises IntegrityError
- Get non-existent agent returns None
- Empty database list returns empty list
- Task history on agent with no tasks returns empty

Errors:
- Task with invalid session_id raises (FK constraint)
- CASCADE: deleting agent deletes tasks
