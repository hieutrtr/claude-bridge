# M14.T3: Wire Poller into MCP Server + Delivery Retry

## Refs
- Architecture: plan/architecture/bridge-mcp.md, Delivery Engine section
- Task def: plan/implementation/phase-4-tasks.md, Task 4.6
- Modules: src/claude_bridge/mcp_server.py, src/claude_bridge/telegram_poller.py
- Dependencies: M14.T1, M14.T2

## What This Task Does
Start poller thread when MCP server initializes. Add delivery retry: unacknowledged inbound messages reset to pending after 3s timeout, up to 5 retries.

## Test Plan
Happy path:
- MCP server starts with poller thread
- Unacknowledged message retried after timeout
- Max retries reached → mark failed

Edge cases:
- Poller handles server shutdown cleanly
