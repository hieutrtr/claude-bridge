## What This Task Does
Test the fallback watcher: detects dead PIDs, handles timeouts, reports unreported tasks.

## Key Architecture Constraints
- Agent state machine: agent always returns to "idle" after task completion
- Task gets the error status (failed/timeout), NOT the agent
- Agent state: running → idle (always), never running → failed

## Test Plan
Happy path:
- Dead PID with result file: marks task done, agent idle
- Reports unreported tasks

Edge cases:
- Dead PID without result file: marks task failed, agent idle
- Timeout: kills process, marks task timeout, agent idle
- No running tasks: exits silently

## Gaps in Existing Code
- watcher.py sets agent state to "failed"/"timeout" — MUST be fixed to "idle"
