## What This Task Does
Background thread that polls Telegram getUpdates, stores messages in inbound_messages, and sends pending outbound messages.

## Key Architecture Constraints
- Long polling (timeout=30s)
- Offset stored in poller_state (survives restart)
- Validates chat_id against access.json allowlist
- Bot token from env or config
- Thread starts/stops cleanly
- Sends pending outbound messages each cycle

## Test Plan
Happy path:
- Poll returns updates, stores in inbound_messages
- Offset advances after processing
- Outbound messages sent via Bot API
- Allowlisted user's message accepted

Edge cases:
- Empty poll (no updates)
- Poll timeout (normal, retry)
- Outbound send fails (increment retry)

Errors:
- Invalid bot token
- Non-allowlisted user rejected
- Network error during poll (retry)
