# M11.T5: Team Result Aggregation

## Refs
- Task def: plan/implementation/phase-3-tasks.md, Task 3.5
- Modules: src/claude_bridge/on_complete.py
- Dependencies: M11.T3 (team dispatch), M11.T1 (parent_task_id)

## What This Task Does
When a sub-task completes, on-complete.py checks if all sibling sub-tasks for the same parent are done. If so, marks the parent team task as done with aggregated cost and summary. Also prints a team completion report.

## Key Architecture Constraints
- on-complete is called per-task, not per-team
- Must detect sub-task vs standalone task (check parent_task_id)
- Total cost = sum of all sub-task costs (lead cost tracked separately)
- Don't auto-dispatch aggregation to lead — just mark parent done and summarize

## Test Plan
Happy path:
- Sub-task completes, parent not yet done, no action
- Last sub-task completes, parent marked done with aggregated cost
- Completion report includes all sub-task summaries

Edge cases:
- Sub-task with no parent (standalone, no-op)
- Some sub-tasks failed (parent still marked done but notes failures)

Errors:
- Parent task doesn't exist (shouldn't happen, but handle gracefully)
