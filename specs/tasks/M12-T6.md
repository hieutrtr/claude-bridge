# M12.T6: Task Completion Notifications

## Refs
- Modules: src/claude_bridge/db.py, src/claude_bridge/on_complete.py, src/claude_bridge/notify.py (new)
- Dependencies: M12.T1 (channel columns on tasks)

## What This Task Does
When a task completes, on_complete.py creates a notification row in SQLite, then attempts direct delivery via Telegram Bot API (urllib). If delivery fails, the notification stays pending for retry.

## Design

### Notifications table
```sql
CREATE TABLE IF NOT EXISTS notifications (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    task_id INTEGER REFERENCES tasks(id),
    channel TEXT NOT NULL,
    chat_id TEXT NOT NULL,
    message TEXT NOT NULL,
    status TEXT DEFAULT 'pending',  -- pending, sent, failed
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    sent_at TIMESTAMP
);
```

### notify.py
- format_completion_message(task) → str
- send_telegram(bot_token, chat_id, message) → bool (urllib)
- deliver_notification(db, notification_id) → bool

### Bot token source
Read from env var TELEGRAM_BOT_TOKEN, or from ~/.claude-bridge/config.json
