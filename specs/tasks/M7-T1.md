## What This Task Does
Extend the tasks table to support queued tasks. Add 'queued' status and position column for ordering.

## Key Architecture Constraints
- New status value: 'queued' (between pending and running)
- Position column: INTEGER, nullable, only set for queued tasks
- Migration must be backward-compatible (existing data preserved)
- Schema init must handle upgrade from Phase 1

## Test Plan
Happy path:
- Create task with status='queued' and position=1
- Query queued tasks ordered by position
- Multiple queued tasks have sequential positions

Edge cases:
- Existing Phase 1 data still works after schema change
- Position is NULL for non-queued tasks

Errors:
- None expected — additive schema change

## Gaps in Existing Code
- No 'queued' status in current schema
- No position column
- No query for queued tasks by session
