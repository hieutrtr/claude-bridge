## What This Task Does
Verify the project structure scaffold is correct: package is importable, session helpers create proper directory structures, and the CLI entry point is callable.

## Key Architecture Constraints
- Bridge home: `~/.claude-bridge/` (workspaces, db, scripts)
- Workspace per session: `~/.claude-bridge/workspaces/{session_id}/tasks/`
- Agent files: `~/.claude/agents/bridge--{session_id}.md`
- No external pip dependencies (stdlib only)
- `python -m claude_bridge.cli --help` must work

## Test Plan
Happy path:
- Package imports successfully
- `__version__` is defined
- `create_workspace()` creates correct directory structure with metadata.json
- `get_workspace_dir()` returns correct path
- `get_tasks_dir()` returns correct path
- `get_agent_file_path()` returns correct path
- CLI `--help` exits with code 0

Edge cases:
- `create_workspace()` is idempotent (calling twice doesn't error)
- `cleanup_workspace()` handles non-existent directory

Errors:
- `cleanup_workspace()` on missing directory doesn't raise

## Gaps in Existing Code
- `create_workspace()` doesn't verify the metadata.json content in tests
- `cleanup_workspace()` not tested
- No test for CLI entry point being callable
- No test for idempotent workspace creation
