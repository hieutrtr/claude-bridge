# P1.T1: Bundle channel server into Python package

## Refs
- Task def: plan/implementation/packaging-tasks.md, P1-T1
- Modules: channel/server.ts, channel/lib.ts, pyproject.toml
- Dependencies: None (first task)

## What This Task Does
Pre-build channel/server.ts → single JS file, bundle into Python package so `pip install` includes it. User doesn't need to `bun install` in channel/ separately.

## Key Architecture Constraints
- `bun build channel/server.ts --target bun --bundle` → single server.js
- Output: `src/claude_bridge/channel_server/dist/server.js`
- pyproject.toml: `package-data` includes `channel_server/**/*`
- Root package.json: `build` script for the bundle step
- dist/server.js committed to git (pre-built artifact)

## Test Plan
Happy path:
- `bun run build` produces dist/server.js
- dist/server.js runs standalone (exits with missing token, not import error)
- `pip install -e .` includes channel_server in installed package
- `bridge-cli` command works after install

Edge cases:
- Build from clean state (no node_modules in channel/)

Errors:
- Missing bun → build fails with clear message
