## What This Task Does
Test the CLAUDE.md initialization logic: new project gets full scan + purpose, existing CLAUDE.md gets agent context appended. All subprocess calls mocked.

## Key Architecture Constraints
- New project: runs `claude -p` with scan prompt that includes project analysis + agent purpose
- Existing CLAUDE.md: runs `claude -p` with append-only prompt
- Uses `--allowedTools "Read,Grep,Glob,Write"` (read-heavy, only writes CLAUDE.md)
- Uses `--output-format json` for structured result
- 120s timeout
- Graceful failure: returns error dict, never raises

## Test Plan
Happy path:
- New project: calls claude with correct scan prompt containing purpose
- Existing CLAUDE.md: calls claude with append prompt
- Returns success dict with message and cost

Edge cases:
- claude CLI not found: returns error dict (not exception)
- Subprocess timeout: returns error dict
- Non-zero exit code: returns error dict
- JSON parse failure on stdout: still returns success

Errors:
- Never raises exceptions — always returns dict
