## What This Task Does
Test the create-agent CLI command end-to-end: validates inputs, derives session identity, generates agent .md file, creates workspace, registers in SQLite. CLAUDE.md init is tested separately in M2.

## Key Architecture Constraints
- Session ID: `{name}--{project_basename}` (double-dash separator)
- Agent .md written to `~/.claude/agents/bridge--{session_id}.md`
- Agent .md has YAML frontmatter: name, tools, model, isolation: worktree, memory: project, Stop hook
- Workspace at `~/.claude-bridge/workspaces/{session_id}/tasks/`
- Agent state starts as 'created'
- Name validation: alphanumeric + hyphens, max 30 chars, no double-dash

## Test Plan
Happy path:
- create-agent produces correct output message
- Agent row exists in SQLite with correct fields
- Agent .md file written with correct frontmatter and body
- Workspace directory created with metadata.json

Edge cases:
- Duplicate agent name returns error (exit code 1)
- Agent .md contains all required frontmatter fields

Errors:
- Missing project dir returns error
- Invalid agent name returns error
- Non-existent project dir returns error

## Gaps in Existing Code
- cli.py cmd_create_agent calls init_claude_md which spawns a subprocess — needs mocking in tests
- No test for the full CLI flow (argparse → handler → DB + files)
- delete-agent silently kills running tasks — should error (fix in M1.T4)
