## What This Task Does
Add `team-dispatch <team_name> "<prompt>"` command. Dispatches to the lead agent with an augmented prompt containing team context, teammate info, and instructions to decompose via bridge-cli.

## Key Architecture Constraints
- Parent task created with task_type='team'
- Lead agent's prompt augmented with team context
- Lead dispatches sub-tasks via Bash tool calling bridge-cli
- Sub-tasks linked via parent_task_id
- Reuses existing dispatch logic (queue-on-busy)

## Design: Augmented Prompt

The lead agent receives:
```
TEAM TASK
=========
{original prompt}

TEAM CONTEXT
============
You are the lead of team '{team_name}'.
Your teammates:
- {member_name}: {purpose} (session: {session_id})

To dispatch sub-tasks to teammates, use:
  bridge-cli dispatch {member_name} "{sub-task prompt}"

To check teammate status:
  bridge-cli status {member_name}

INSTRUCTIONS
============
1. Decompose the task into sub-tasks for your teammates
2. Dispatch each sub-task
3. Monitor progress
4. When all done, aggregate results and provide a summary
```

## Test Plan
Happy path:
- team-dispatch creates parent task with type='team'
- Lead agent receives augmented prompt with team context
- spawn_task called with correct arguments

Edge cases:
- Team doesn't exist (error)
- Lead agent is busy (task queues)

Errors:
- Non-existent team name
