## What This Task Does
Test the Stop hook handler: parses result JSON, updates SQLite, prints report.

## Key Architecture Constraints
- Called with `--session-id` arg by Claude Code Stop hook
- Finds running task for session, parses result file
- Updates task: status (done/failed), summary, cost, duration, completed_at
- Updates agent: state → idle, increment total_tasks
- Prints report to stdout

## Test Plan
Happy path:
- Successful task: parse JSON, update to done, print success report
- Failed task (is_error=true): update to failed, print error report

Edge cases:
- Missing result file: marks task as failed
- Empty result file: marks task as failed
- Malformed JSON: marks task as failed
- No running task for session: exits silently

Errors:
- Never raises — handles all errors gracefully

## Gaps in Existing Code
- on_complete.py uses sys.path.insert hack — should use -m invocation
- watcher.py sets agent state to "failed"/"timeout" — should be "idle" (fix in M4.T2)
