## What This Task Does
Implement bridge_get_messages and bridge_acknowledge MCP tools. Get messages marks them delivered, acknowledge confirms processing. Unacknowledged messages retry after 3s.

## Test Plan
Happy path:
- bridge_get_messages returns pending, marks delivered
- bridge_acknowledge marks acknowledged
- Subsequent get_messages skips acknowledged

Edge cases:
- No pending messages
- Acknowledge nonexistent message

Errors:
- Invalid message_id
