## Refs
- Architecture: plan/architecture/bridge-mcp.md, Message Queue section

## What This Task Does
Create `MessageDB` class with SQLite schema for inbound_messages, outbound_messages, and poller_state tables. Separate DB from bridge.db to avoid write contention.

## Key Architecture Constraints
- Separate messages.db (not bridge.db)
- WAL mode
- Status transitions: pending → delivered → acknowledged / failed
- Retry tracking: retry_count, max_retries
- Poller state: telegram offset persistence

## Test Plan
Happy path:
- Create inbound message, read it back
- Create outbound message, read it back
- Get/set poller state
- Get pending inbound messages
- Get pending outbound messages
- Mark inbound as delivered, acknowledged
- Mark outbound as sent

Edge cases:
- No pending messages returns empty list
- Mark nonexistent message (no-op)

Errors:
- DB path auto-creates directory
