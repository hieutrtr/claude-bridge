## What This Task Does
Instead of rejecting when agent is busy, queue the task with a position.

## Key Architecture Constraints
- Idle agent: dispatch immediately (no change)
- Busy agent: insert with status='queued', position=next_in_line
- Print queue position to user

## Test Plan
- Dispatch to idle: immediate (existing behavior)
- Dispatch to busy: queued at position 1
- Dispatch third task: queued at position 2
- Output includes queue position

## Gaps in Existing Code
