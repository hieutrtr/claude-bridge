# Architecture — Claude Bridge

## Core Concepts

### Agent = Claude Code Process

Mỗi agent về mặt kỹ thuật là một Claude Code process:

```bash
claude --project ~/my-app \
       --print \
       --dangerously-skip-permissions \
       -p "System prompt từ profile..."
```

Bridge wrap process này, pipe messages từ Telegram vào stdin, đọc stdout gửi về chat.

### Two Agent Modes

**Mode 1 — Headless Task Runner**
```
User → Telegram → Bridge → spawn claude --print → done → Telegram
```
- Fire and forget
- Claude tự spawn subagents bên trong (up to 5 simultaneous)
- Phù hợp: tasks rõ ràng, không cần nhiều vòng trao đổi

**Mode 2 — Persistent Conversation Agent**
```
User → Telegram → Bridge → forward vào session sống trong tmux
Claude → reply → Bridge → Telegram
```
- Session giữ context lâu dài
- Phù hợp: debug phức tạp, review code, tasks cần nhiều vòng
- Chạy trong tmux để persist khi terminal đóng

### Transport Layer

Dùng **Claude Code Channels** (MCP):

```
Channel Plugin (local process)
    ↕ stdio → MCP SDK
Claude Code Session
    ↕ poll outbound (không expose port)
Telegram Bot API
```

Không có inbound ports, không webhook public. An toàn hoàn toàn.

## Profile System

### Profile là nguồn sự thật duy nhất

```
profile.yaml  →  Bridge đọc và quản lý
    ↓ generates
CLAUDE.md     →  Agent đọc và làm việc
    ↓ generates  
settings.json →  Hooks config
    ↓ generates
commands/     →  Skills
```

Khi profile thay đổi → Bridge re-generate tất cả.

### Profile Lifecycle

```
[Onboarding 3 câu hỏi]
        ↓
[Version 1 — minimal, đủ dùng]
        ↓
[Làm việc — Bridge quan sát]
        ↓
[Signals tích lũy trong session_logs]
        ↓
[Trigger enhance — auto hoặc /enhance]
        ↓
[Bridge đề xuất → user approve 1 tap]
        ↓
[Profile update → CLAUDE.md re-generated]
        ↓
[Version N — ngày càng mạnh hơn]
```

### Enhancement Triggers

1. **Micro (sau mỗi session)**: Quick wins, 1-2 suggestions
2. **Macro (/enhance)**: Deep analysis nhiều sessions, suggestions batch
3. **Real-time**: User correct agent → Bridge hỏi ngay có muốn save không

### Enhancement Principles

- **Passive by default**: Không tự ý thay đổi, luôn hỏi
- **One tap to apply**: Không yêu cầu gõ text
- **Hard rules bất khả xâm phạm**: User-defined rules không bao giờ bị evolve away

## CLAUDE.md Layer System

```
~/.claude/CLAUDE.md                    ← Global (user quản lý)
~/projects/my-app/CLAUDE.md            ← Project (Bridge generate)
~/projects/my-app/src/auth/CLAUDE.md   ← Sub-dir (Bridge generate khi cần)
```

Bridge quản lý project-level trở xuống.

## Hooks System

Hooks là "luật cứng" — không cạnh tranh attention với context window.

### Hook Types dùng trong Bridge

| Hook | Trigger | Dùng cho |
|---|---|---|
| `SessionStart` | Session mới | Load dynamic context, inject state |
| `PreToolUse[Bash]` | Trước bash | Block dangerous commands |
| `PostToolUse[Write]` | Sau write | Chạy linter, formatter |
| `Stop` | Agent sắp dừng | Kiểm tra task thực sự xong chưa |
| `PreCompact` | Trước compaction | Update dev docs để không mất context |

### Permission Relay Hook

Khi agent cần permission → hook intercept → gửi về Telegram:

```
PreToolUse → Bridge → Telegram inline keyboard
                        [✅ Approve] [❌ Deny] [👁️ Preview]
                      ← user tap
Bridge → exit 0 (allow) hoặc exit 2 (block)
```

## Session Persistence

Vấn đề: Claude Code context compaction làm mất context.

Giải pháp của Bridge:
1. **Dev docs**: `plan.md`, `context.md`, `tasks.md` persist ngoài conversation
2. **PreCompact hook**: Tự update dev docs trước khi compact
3. **SessionStart hook**: Load dev docs vào context khi resume
4. **tmux wrapper**: Session không chết khi terminal đóng

## Security Model

- Channel plugin chạy local, poll outbound — không expose ports
- Sender allowlist: chỉ Telegram user ID đã pair mới gửi được
- `--channels` flag phải explicit mỗi session
- Hard rules không thể bị override bởi agent
- Session logs không chứa sensitive data
